## Function Signature

```python
image.rotate(angle: float) -> pxt.Image
```

## Description

Transform visual perspective through intelligent rotation, where every degree of change opens new possibilities for understanding. Rotation is not merely geometric transformation—it's **perspective liberation** that adapts visual data to correct orientations, augment datasets, and reveal hidden insights.

In the realm of computer vision and AI, image orientation can determine the difference between recognition and confusion. This function embodies Pixeltable's philosophy that **spatial transformations should be as effortless as they are powerful**, enabling data scientists to correct, augment, and explore visual data with declarative simplicity.

## Parameters

<ParamField path="angle" type="float" required>
  The angle to rotate the image, in degrees. Positive angles rotate counter-clockwise, negative angles rotate clockwise.
  
  **Common Use Cases:**
  - `90`, `180`, `270` - Standard orthogonal rotations for orientation correction
  - `45` - Diagonal rotation for artistic effects or data augmentation
  - `0.5`, `1.2` - Micro-rotations for subtle image corrections
  - `-15`, `-30` - Clockwise rotations for perspective adjustment
  - `random.uniform(-10, 10)` - Random rotation for data augmentation pipelines
</ParamField>

## Returns

<ResponseField name="rotated_image" type="pxt.Image">
  A new image rotated by the specified angle. The image dimensions may increase to accommodate the rotated content, with transparent or black pixels filling any empty areas.
</ResponseField>

## Examples

### Orientation Correction Pipeline

Fix image orientations automatically in your data pipeline:

```python
import pixeltable as pxt

# Create table for images with orientation issues
photos = pxt.create_table('photo_collection', {
    'id': pxt.String,
    'raw_photo': pxt.Image,
    'rotation_needed': pxt.Float  # Degrees to rotate for correction
})

# Add computed column that applies rotation correction
photos.add_computed_column(
    corrected=photos.raw_photo.rotate(photos.rotation_needed)
)

# Insert photos with their required corrections
photos.insert([
    {'id': 'portrait_001', 'raw_photo': '/photos/sideways.jpg', 'rotation_needed': 90},
    {'id': 'landscape_002', 'raw_photo': '/photos/upside_down.jpg', 'rotation_needed': 180},
    {'id': 'perfect_003', 'raw_photo': '/photos/correct.jpg', 'rotation_needed': 0}
])

# All photos are now properly oriented in the corrected column
results = photos.select(photos.id, photos.corrected).collect()
```

## Technical Implementation Notes

### Rotation Mechanics

- **Counter-clockwise**: Positive angles (0° to 360°)
- **Clockwise**: Negative angles (-360° to 0°)
- **Full rotation**: 360° returns to original orientation
- **Dimension changes**: Rotated images may have larger dimensions to fit rotated content

### Performance Characteristics

- **Incremental Computation**: Rotation operations are cached and only recomputed when source images or angles change
- **Memory Efficiency**: Optimized memory usage for intermediate rotation buffers
- **Batch Processing**: Multiple rotations on the same image are processed efficiently

### Image Quality Preservation

The rotation operation uses high-quality interpolation to preserve image clarity:

```python
# PIL equivalent (manual approach):
from PIL import Image
img = Image.open('/path/to/image.jpg')
rotated = img.rotate(45, expand=True)

# Pixeltable equivalent (declarative):
images.add_computed_column(
    rotated=images.raw_image.rotate(45)
)
```
